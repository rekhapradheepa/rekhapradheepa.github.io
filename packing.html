PACKING

Packing involves placing the items into the appropriate sized box with the right packaging materials to ensure they reach their final destination undamaged. To reduce shipping costs, packing should keep an order within the smallest number of boxes and use the right type of packing material(s) for each product.

In the packing area, shipping cartons are build. Packing will retrieve product from previous area’s like Picking and Unit VAS when unit vas is done before packing.

Packing Process

There are different kinds of holders that can be brought to the Packing area: totes, shipping cartons, trolley’s and shelve units. All of these holders will need to be repacked into shipping cartons.

After the packing process, the shipping files will need to be updated and mostly the shipping label and carton content label needs to be printed. When the holder that comes to the packing area is already a shipping carton, then the shipping label does not need to be printed anymore. By use of parameters the system will determine if carton content label needs to be printed or not.

For the Digital Cartons, the carrier shipping label will be printed. For both the Apparel and FTW Digital Cartons, the Carton Content Label will not be printed. This is controlled using a parameter

For digital cartons, after Packing and/or the optional VAS activity is completed, user will be directed to Weight & Size processing screen where the user will perform weighing and update the same as required.

Let us know about process flow

Once the holder is scanned in packing screen, it checks whether the holder is attached to single shipping carton or multiple shipping carton.UPC checking will be done during packing process.Once packer confirms that he packed the shipping carton,labels will get printed.During packing process, exceptions such as shortage, overage, stop shipment might occur.
